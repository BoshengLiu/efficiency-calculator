## 1. 后端 [server.py](d:\project\automation-value-center\server.py)

- **POST /api/track**：只接受 `application/json`，校验 `event`（page_view / calculator_use / consult_click）和 `ts`，将记录追加到 `data/events.jsonl`，返回 204。
- **GET /api/stats**：支持查询参数 `from`、`to`、`event`、`tool_id`、`limit`。返回 JSON：
  - **by_day**：按日的 `page_view` / `calculator_use` / `consult_click` 数量
  - **by_event**：各事件总次数
  - **by_tool**：各工具（tool_id）次数
  - **recent_consult**：最近 N 条咨询点击（含 ts、source、last_tools）
- 首次写入前会自动创建 `data/` 目录。

## 2. 前端埋点 [js/utils/analytics.js](d:\project\automation-value-center\js\utils\analytics.js)

- **track(event, data)**：用 `sendBeacon` 或 `fetch` 上报到 `/api/track`；`calculator_use` 按 5 秒节流，并写入 `sessionStorage` 的 `avc_last_tools`。
- **trackPageView(overrides)**：根据当前 path 解析 `tool_id`（`/tools/xxx/`）并上报 page_view。
- **trackConsultClick(overrides)**：上报 consult_click，带上 `source` 和 `last_tools`，上报后清空 `last_tools`。

## 3. 页面接入

- **首页**：引入 `analytics.js`，在 `main.js` 的 `init()` 中调用 `Analytics.trackPageView()`，在「立即咨询」点击时调用 `Analytics.trackConsultClick()`。
- **6 个工具页**（cost/time/error/feasibility/ai-potential/document-parser-demo）：均引入 `analytics.js`，在 DOMContentLoaded 时调用 `Analytics.trackPageView()`。
- **6 个工具 JS**：在用户触发计算的地方（debounced 回调或选项/文件处理）调用 `Analytics.track('calculator_use', { tool_id: '…' })`，tool_id 与目录名一致。

## 4. 统计看板 [admin/stats.html](d:\project\automation-value-center\admin\stats.html)

- 提供日期范围筛选和「刷新」按钮，请求 `/api/stats` 并展示：
  - **按日统计**：每日 page_view / calculator_use / consult_click
  - **事件汇总**：各事件总次数
  - **按工具统计**：各 tool_id 次数
  - **最近咨询点击**：时间、来源、使用过的工具（last_tools），便于优先跟进「热」线索。

使用方式：运行 `python server.py`，访问首页与各计算器会产生埋点；查看统计与热线索可打开 **http://localhost:8000/admin/stats.html**。

```markdown
- **页面浏览 (page_view)**：打开首页或任意工具页时自动上报。

数据仅存储在本地，不依赖第三方统计平台。

### 查看统计看板

1. 确保使用 `python server.py` 启动（统计 API 仅在此服务器下可用）。
2. 在浏览器中打开 **统计看板**：  
   **http://localhost:8000/admin/stats.html**
3. 看板提供：
   - **按日统计**：每日的页面浏览、计算器使用、咨询点击数量。
   - **事件汇总**：各事件类型的总次数。
   - **按工具统计**：各计算器（工具）的访问/使用次数。
   - **最近咨询点击**：时间、来源页面、使用过的工具列表，用于优先跟进高意向访客。

可设置「从」「到」日期后点击「刷新」筛选时间段。

### 原始数据与 API

- **存储位置**：`data/events.jsonl`（每行一条 JSON，由 server.py 在首次上报时自动创建 `data/` 目录）。
- **上报接口**：`POST /api/track`（前端埋点脚本 `js/utils/analytics.js` 调用）。
- **查询接口**：`GET /api/stats`，支持查询参数 `from`、`to`、`event`、`tool_id`、`limit`，返回聚合后的 JSON，供看板或自行脚本/Excel 分析使用。

部署到公网时，建议对 `/api/stats` 做简单鉴权（如 query 密钥或 Basic Auth），避免他人随意查看。
```
